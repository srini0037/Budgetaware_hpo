Phase 1 – What Has Been Done
Logistic Regression · Baseline vs HPO

In Phase 1, Logistic Regression was trained and evaluated on three datasets to understand where hyperparameter optimization actually provides value. The focus was on comparing baseline performance (default hyperparameters) with HPO-tuned performance using F1 macro score.

Key outcomes
  • • Breast Cancer and Adult datasets showed almost no improvement from HPO, suggesting that Logistic Regression is relatively insensitive to hyperparameters on these datasets.
  • • The German Credit (credit-g) dataset showed a clear performance gain from HPO, making it a good main case study for Logistic Regression.

Interpretation

Breast Cancer and Adult are effectively “solved” by Logistic Regression without much help from hyperparameter tuning. In contrast, the credit-g dataset benefits substantially from HPO (ΔF1 ≈ +0.058), making it a meaningful setting to study budget-aware HPO strategies for Logistic Regression in later phases.

Next Phases – Budget-Aware HPO & Meta-Learning
Roadmap
How Phase 1 feeds into the rest

Phase 1 establishes where HPO matters for Logistic Regression and provides a solid, experiment-based justification for using credit-g as a main case study. The next phases build on this by introducing budget-aware strategies and then meta-learning for automatic resource allocation.

Phase 2 – Budget-Aware HPO
Implementation

Design and compare low-budget vs high-budget HPO strategies.

  • Low-budget HPO: fewer trials, narrower search space, fast evaluations.
  • High-budget HPO: more trials, wider search space, full evaluations.
  • Budget–performance curves: evaluate performance at time/trial budgets (e.g. 5s, 10s, 20s, 40s, 80s, 120s, ...).
  • Crossover point: find where the high-budget strategy begins to outperform the low-budget strategy consistently.
Phase 3 – Meta-Features & Meta-Dataset
Meta-Knowledge

Generalise budget-aware behaviour across multiple datasets.

  • Extract dataset-level meta-features (e.g. size, dimensionality, class imbalance, etc.).
  • Compute budget–performance curves and crossover points for each dataset.
  • Build a meta-knowledge base: meta_dataset.csv with meta-features + crossover statistics.
Phase 4 – Meta-Learning for Smart Resource Allocation
Meta-Model

Train a meta-model that chooses how to spend HPO budget on a new dataset.

  • Input: dataset meta-features.
  • Outputs:
    • Recommended HPO budget (e.g. number of trials or time budget).
    • Recommended HPO strategy (low-budget vs high-budget, or others).
    • Estimated crossover threshold.
  • This becomes a “smart resource allocator” for hyperparameter optimization.
